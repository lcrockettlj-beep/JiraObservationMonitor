# Stakeholder Demonstration Brief - Production Candidate v2

## Suggested demonstration flow

1. Open Command Centre and explain estate health, runtime state, source state, alerts and discovery queue.
2. Open Estate and show monitored vs discovered sites.
3. Select a site in Estate and open the Site Workspace from the detail panel.
4. Open Admin and show discovery governance, named access and runtime/source health.
5. Show that compatibility `/api` routes remain intentionally retained while the active frontend uses the consolidated workspace model.

## Key message

JOM is now operating as a read-only Atlassian operational intelligence platform with production-candidate frontend workspaces and validated backend/operator contracts.
