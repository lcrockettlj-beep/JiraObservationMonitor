# Stakeholder Demo Brief

## Recommended Demo Flow
1. Open Command Centre and explain current estate posture.
2. Show readiness strip and live source-backed runtime status.
3. Open Estate and select a monitored site.
4. Open the Site Workspace from the Estate detail panel.
5. Open Admin and show discovery, named access, runtime and source intelligence.
6. Explain that the platform is read-only and designed for operational truth, not UI-generated assumptions.

## Key Talking Points
- JOM provides a single operational view of the Atlassian estate.
- The platform separates executive overview, estate investigation, governance control and site-level drilldown.
- The UI is now standardised on one Atlassian-style visual system.
- The current release candidate focuses on reliable visibility and operational confidence.

## Avoid Overclaiming
- Do not describe planned export placeholders as completed exports.
- Do not describe Admin controls as write-back actions unless write capability is explicitly added later.
- Do not present synthetic values; only discuss source-backed values visible in the app.
