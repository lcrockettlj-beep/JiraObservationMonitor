JIRA OBSERVATION MONITOR
MANUALS INDEX
=============

This pack contains project-specific manuals for the main languages and template technologies used by Jira Observation Monitor.

Included manuals:

1. python_project_manual.txt
   - explains how Python works in this project
   - backend flow
   - service modules
   - _build_data()
   - routes
   - drilldown engine
   - Python debugging and safe editing

2. html_project_manual.txt
   - explains how HTML templates are structured
   - nav pattern
   - hero pattern
   - page structures
   - relationship between HTML, CSS, and JS
   - safe template editing

3. css_project_manual.txt
   - explains app.css, home.css, detail.css
   - theme variables
   - nav styling
   - site card styling
   - responsive layout
   - safe CSS editing and debug flow

4. javascript_project_manual.txt
   - explains theme.js and current JavaScript usage
   - localStorage theme persistence
   - initToggle workflow
   - safe JS editing
   - JS troubleshooting

5. jinja_template_manual.txt
   - explains how Jinja works in this project
   - variable output, loops, conditionals, macros
   - how Python data reaches templates
   - Jinja debugging and safe editing rules

Suggested reading order for a new developer:
1. jira_observation_monitor_handover_manual.txt
2. python_project_manual.txt
3. html_project_manual.txt
4. jinja_template_manual.txt
5. css_project_manual.txt
6. javascript_project_manual.txt

Suggested reading order for you as the project owner:
1. python_project_manual.txt
2. html_project_manual.txt
3. css_project_manual.txt
4. javascript_project_manual.txt
5. jinja_template_manual.txt

The goal of these manuals is not generic theory.
The goal is to explain how THESE technologies work in THIS project.
